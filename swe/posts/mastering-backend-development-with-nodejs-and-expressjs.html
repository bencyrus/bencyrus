Your Guide to Mastering Backend Development: Key Concepts Beyond Any Specific Tool

Published on Oct 9, 2023 at 10:00 AM

After years of crafting roadmaps and guiding new developers, I've put together an easy-to-follow backend roadmap. This isn't just about NodeJS or ExpressJS. It's a full roadmap showing you how backend systems really work. You'll see all the main parts and processes, without any extra fluff to confuse you.

It's worth mentioning that this roadmap isn't for complete beginners to coding. Before embarking on this journey, you should have a basic grasp of JavaScript and, at the very least, know what NodeJS and ExpressJS are. By "know," I simply mean being aware of their existence and basic purpose as well as having read or written a few lines of code in them.

Each section of the roadmap breaks topics down based on subject matter, making them easily searchable. As a method of measurement, when you're learning each topic, challenge yourself with this question: "Can I teach this to someone else?" If the answer is yes, then you've truly grasped the concept.
It is also essential to note that many topics you would see in regular tutorials such as database integration and anything that goes beyond the concepts and basic frameworks are not included in this roadmap. This is because those are typically the areas that cause a lot of errors and confusion for new developers. Instead, you can use local JSON or CSV files for data storage. This helps you keep your attention on the main ideas, and reduce the risk of getting overwhelmed.

Because there's so much to learn in backend development, I've only included the points you need to know to have a complete mental picture of a backend system. This way, you will understand almost any backend system, regardless of the technology it uses. With this strong base, you'll be able to learn any other details you need later on.

By the end of this roadmap, you'll be equipped to craft dynamic applications where users can read, write, and delete data with confidence.

Phase 5: Apply Knowledge by Building a Project

By now, you should be armed with the tools to construct a backend. The goal of this phase is to translate that knowledge into practice. Don't fret about the complexity of your projectâ€”the aim is to consolidate what you've learned.

Objective: Construct a project that applies basic CRUD (Create, Read, Update, Delete) operations using NodeJS, ExpressJS, and JSON files as your makeshift database.

Project Guidelines:
  1. Scope: The project's core functionality should involve CRUD operations. Whether you're building a system to manage a list of names, todos, or even just a "Hello World" message board, the main thing is to apply what you've been studying.
  2. Data Models: Define clear data models for your application. For instance, if you're making a simple user management system, a user might have attributes like `id`, `name`, and `email`.
  3. Endpoints:
    • GET Request: Retrieve data. For example, retrieving all users or a specific user.
    • POST Request: dd new data. Like adding a new user.
    • PUT Request: Update existing data. Maybe updating a user's name or email.
    • DELETE Request: Delete data, such as removing a user from the system.
  4. Storage:
    • Store your data as JSON files in a `/data` directory.
    • For each new record, generate a unique ID. Tip: Fetch all items, identify the highest ID, then add one (+1) to determine the new record's ID.
  5. Tools: Use Postman or a similar platform to test your API endpoints.

Key Takeaway: Remember, the essence of this phase is application. It's through practice that we truly consolidate learning. While you might feel like you're in unfamiliar territory, trust the process and use documentation and online resources to guide you.

Phase 7: Building a Fully Functioning Application

By reaching this phase, you've acquired a solid backend foundation. Now, it's time to combine your backend prowess with frontend skills to create a comprehensive, functional app.

Use the project you designed in the first phase. Do any modifications required based on what you have learned.

Objective: To build an end-to-end application that showcases both frontend and backend capabilities.

Steps to Follow:

  1. User Story: Begin by visualizing the user's journey. Think about what functionalities and pages the user will interact with. Write a user story.
  2. Specification Document: Write a specification document that outlines the app's features and functionalities. This document should include the following:
    • App's name
    • App's description
    • App's features
    • App's pages
    • App's user flow
    • App's data models
    • App's endpoints
    • App's database tables
    • App's UI
  3. Sequence Diagram: Write a sequence diagram (or update the previous one you created in the first phase) to show the app's flow.
  4. Entity Relationship Diagram (ERD): Create an ERD (or update the previous one you created in the first phase) to visualize how different entities in your app relate to one another, especially in the context of your database.
  5. Backend Prototype with Dummy Data: Begin your backend development, but instead of integrating with a real database, use dummy data. This is like creating an API mock to understand the flow better.
  6. Frontend Development: You can use anything for the frontend. Raw HTML, CSS, and JavaScript, or a frontend framework like ReactJS, VueJS, or AngularJS. Use the API endpoints as if they actually work.
  7. Integrating Backend with Real Data:
    • Substitute the dummy data in your backend with real data storage, preferably using a JSON file as your database (as you've practiced before).
    • Ensure CRUD operations function seamlessly.
  8. Finish the Frontend: Update the frontend to use the real API endpoints and use the data and perform operations.
